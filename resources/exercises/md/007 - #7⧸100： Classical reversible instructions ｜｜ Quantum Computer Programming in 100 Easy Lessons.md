# Exercises

This exercise is not related to the preceding lesson, but it will be helpful for the upcoming few.  
It refers back to the exercise from Episode 5.

Make a new version of your Probabilistic Computer Simulator.  The input  
should be as before, but the output should be different.  Instead of *simulating* the code using  
(pseudo)randomness and printing out all the final bit values, your code should *compute the true,  
exact probability* that the final bit values are $000 \cdots 0$ (all $n$ bits 0). This will be a rational number,  
like maybe “3/8”.

I expect you’ll find it would be majorly difficult to tolerate  
$n = 100,000$.  In fact, it is OK if your code only handles $n$ up to 10.
